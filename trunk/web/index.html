HTML Include

Version 0.10

Overview

HTML Include solves the problem that the Hypertext Markup Language doesn't know a command for including other files into a source file. So if use some pieces of source code (i. e. Hypertext fractions) in more than one file than you need to copy the piece into every single file. And if you change something, than you have to manually modify all files (that is, if you do not have any type of special editor which handles this for you). But now you can delegate this task to HTML Include which checks the given source file for certain Tags (special-styled comments) and replace the content of the include area, if the content in the associated include file was changed.
Taken from the HTML Include documentation.

If you want to see an example, just examine this page in any text editor and look for comments with the text "INCLUDE" and "ENDINC" in them. You can also see the content fo the included files and the run-script for HTML Include.

Get it from BerliOS!

The HTML Include project page is hosted at BerliOS: http://developer.berlios.de/projects/htinc/. There you can find further information and download the latest release.

How to retrieve HTML Include from SVN

HTML Include is developed using the Subversion repository of BerliOS (therefore you cannot find any files in the CVS-area of this project, in case you wondered). Subversion is a version control system designed to be superior and eventually replace CVS. Their home page is located at http://subversion.tigris.org in case you want to find out more about it.

If you want to retrieve the latest developer version of HTML Include, here is the instruction how to get it from the Subversion repository:

  • Type the following:
    svn checkout svn://svn.berlios.de/htinc
  • After initial checkout, you can change into this directory and execute SVN commands without the URL. For example:
    svn status
Taken from BerliOS' Subversion (SVN) HowTo.
